## MODULE 1: KEYWORD RESEARCH ENGINE
*Source: aaron-he-zhu/seo-geo-claude-skills — keyword-research*

### Trigger
User asks for keyword ideas, keyword research, what to rank for, or which terms to target.

### 8-Phase Keyword Research Workflow

**Phase 1: SCOPE**
Clarify before starting:
- What is the product/service/topic?
- Who is the target audience?
- What is the domain authority (DA) level? (0-20 = new, 20-40 = growing, 40+ = established)
- What geography/country are we targeting?
- What language?
- What are the primary business goals? (traffic, leads, sales)

**Phase 2: DISCOVER**
Extract four keyword buckets:
1. **Core terms** — Direct product/service descriptors (e.g., "IPTV service", "IPTV subscription")
2. **Problem-focused** — Pain points the audience has (e.g., "cable TV too expensive", "how to watch foreign channels")
3. **Solution-focused** — How they search for answers (e.g., "best IPTV provider 2026", "IPTV vs cable TV")
4. **Audience-specific** — Niche terminology, role-based language, community jargon

**Phase 3: VARIATIONS**
Generate modifier expansions:
- Qualifiers: best, top, affordable, cheap, premium, reliable
- Question formats: how to, what is, why does, when to
- Long-tail patterns: "[term] for [audience]", "[term] in [location]", "[term] vs [alternative]"
- Commercial: buy, price, cost, subscription, plan, trial, discount
- Comparison: "[brand] vs [brand]", "alternatives to [brand]"

**Phase 4: CLASSIFY BY INTENT**
| Intent Type | Description | Example |
|-------------|-------------|---------|
| Informational | Learning/research | "what is IPTV" |
| Navigational | Finding a specific site | "IPTV service login" |
| Commercial | Comparing before buying | "best IPTV providers 2026" |
| Transactional | Ready to buy/subscribe | "buy IPTV subscription" |

**Phase 5: SCORE — Opportunity Formula**
```
Opportunity Score = (Search Volume × Intent Value) / Keyword Difficulty

Intent Values: Transactional = 4 | Commercial = 3 | Informational = 2 | Navigational = 1
```
Score interpretation: 100+ = Priority Target | 50-99 = Strong Target | 20-49 = Secondary | <20 = Low priority

**Phase 6: GEO-CHECK**
Identify keywords likely to trigger AI responses:
- Question formats (triggers People Also Ask + AI Overviews)
- Definition queries ("what is...", "X is a type of...")
- Comparison queries ("X vs Y")
- "Best [X] for [Y]" patterns (triggers AI recommendation lists)
- Local intent ("IPTV service near me", "IPTV provider [city]")

**Phase 7: CLUSTER**
Group into topic clusters:
- **Pillar page** (high-volume, broad, commercial intent)
- **Cluster pages** (long-tail, specific, informational)
- **Supporting pages** (FAQ, comparison, review sub-topics)

**Phase 8: DELIVER**
Output format for every keyword recommendation:
```
Keyword: [exact term]
Est. Monthly Volume: [number]
Keyword Difficulty (KD): [0-100]
Opportunity Score: [calculated]
Intent: [type]
GEO Trigger Potential: [High/Medium/Low]
Current Ranking URL: [if applicable]
Recommended Action: [create new / optimize existing / target via FAQ]
```

### Output Requirements
Every recommendation MUST include specific data. NEVER say "target long-tail keywords." Instead:
> "Target 'IPTV subscription 4K streaming' (est. vol: 590/mo, KD: 19) — no DR>40 sites in top 10. Opportunity Score: 93. High GEO trigger potential."

## MODULE 2: SEARCH INTENT ANALYZER
*Source: aaron-he-zhu/seo-geo-claude-skills — serp-analysis*

### Trigger
User wants to understand what type of content to create for a keyword, or why their page isn't ranking.

### SERP Analysis Workflow

**Step 1: Identify the Dominant Content Format**
Analyze top 10 results for:
- Content type: Blog post / Product page / Video / Tool / Category page / Landing page
- Content angle: Beginner guide / Expert deep-dive / Comparison / List / Tutorial / Definition
- Content length signals: Are top results 500 words or 3,000+ words?

**Step 2: Identify AI Answer Patterns**
- Does this keyword trigger an AI Overview / Featured Snippet?
- What content format does the AI Overview use? (paragraph / list / table / step-by-step)
- What source type does Google AI cite? (blog / .gov / .edu / brand site)
- Does it trigger People Also Ask? What questions appear?

**Step 3: Map the Content Gap**
- What do ALL top 10 results cover? (table stakes content)
- What do SOME results cover? (differentiators)
- What does NONE cover? (white space opportunity)
- What questions does the PAA box reveal that no one answers well?

**Step 4: Intent Alignment Matrix**
```
Searcher wants → Content must deliver
├── Information: Direct answer in first 150 words + comprehensive coverage
├── Comparison: Side-by-side comparison tables + clear recommendation
├── Purchase: Pricing, trust signals, CTAs, social proof
└── Navigation: Fast load, clear branding, easy login/access path
```

**Step 5: AI Answer Format Mapping**
| Query Type | Optimal AI Answer Format |
|-----------|--------------------------|
| "What is X" | 25-50 word definition + 134-167 word explanation block |
| "How to X" | Numbered steps, each 1-2 sentences |
| "Best X for Y" | Comparison table + ranked list with specific reasons |
| "X vs Y" | Side-by-side table + summary verdict |
| "Why X" | Direct cause statement + supporting evidence |

## MODULE 3: SEO CONTENT GENERATOR
*Source: aaron-he-zhu/seo-geo-claude-skills — seo-content-writer + AgriciDaniel/claude-seo — seo-content*

### Trigger
User wants to write or generate SEO-optimized content.

### Pre-Writing Requirements Checklist
Before writing ANY content, confirm:
- [ ] Primary keyword (exact match)
- [ ] Secondary keywords (3-5 semantic variants)
- [ ] Target audience (persona)
- [ ] Content intent (informational/commercial/transactional)
- [ ] Target word count (minimum 1,500 words for blog; 800 for landing page)
- [ ] Tone (authoritative / friendly / technical / conversational)
- [ ] Target readability: Flesch Reading Ease 60-70

### SERP Analysis (Before Writing)
1. Identify top 3 ranking pages for the primary keyword
2. Note their content structure, headings, word count
3. Identify what they do NOT cover (your differentiation angle)

### Content Architecture

**Title Formula** (must include primary keyword, under 60 characters):
- How-to: "How to [Achieve Result] with [Method]"
- List: "[Number] Best [Topic] for [Audience] in [Year]"
- Question: "What Is [Topic]? [Benefit Statement]"
- Commercial: "[Primary Keyword]: [Differentiator]"

**Meta Description Formula** (150-160 characters):
```
[Primary benefit + keyword] + [unique differentiator or proof point] + [soft CTA]
```

**Content Structure:**
```
H1: [Primary Keyword + Value Proposition]
├── Opening (150 words): Direct answer to search intent + hook
├── H2: [Secondary topic 1]
│   ├── H3: [Subtopic]
│   └── H3: [Subtopic]
├── H2: [Secondary topic 2]
├── H2: [Secondary topic 3]
├── H2: FAQ Section (AEO)
│   ├── Q: [Question from PAA/searcher intent]
│   └── Q: [Question from PAA/searcher intent]
└── Conclusion: Summary + CTA
```

### 16-Point CORE-EEAT Content Constraints
Apply ALL of these before publishing:

**CORE Constraints:**
1. **C01 — Intent Alignment** *(VETO GATE)*: Content must match the dominant search intent exactly
2. **C02 — Direct Answer First**: Open with answer in first 1-2 sentences
3. **C03 — Heading Hierarchy**: H1 > H2 > H3 (never skip levels)
4. **C04 — Citation Density**: 3-5 specific statistics per 500 words
5. **C05 — Keyword Placement**: Primary keyword in H1, first 100 words, at least one H2
6. **C06 — Semantic Variants**: Include 3-5 related terms naturally throughout
7. **C07 — Internal Links**: 2-5 links to relevant internal pages
8. **C08 — External Links**: 2-3 links to authoritative sources

**EEAT Constraints:**
9. **E01 — Experience**: Include first-person experience markers or direct observation
10. **E02 — Expertise**: Demonstrate subject-matter depth with technical precision
11. **A01 — Authority**: Link to or cite authoritative sources
12. **A02 — External Validation**: Reference industry recognition, credentials, or studies
13. **T01 — Transparency**: Disclose affiliations, update dates, methodology
14. **T02 — Accuracy**: Every claim must be verifiable
15. **T03 — Trustworthiness Signals**: Include author bio, credentials, date
16. **T04 — Disclosure Statements** *(VETO GATE)*: Affiliate/sponsored content must be clearly labeled

### On-Page SEO Final Checklist
- [ ] Title tag: Primary keyword present, 50-60 characters
- [ ] Meta description: 150-160 characters, includes keyword + CTA
- [ ] H1: One per page, includes primary keyword
- [ ] Image alt text: All images have descriptive alt text with keyword where natural
- [ ] Keyword density: Primary keyword 1-2%, avoid stuffing
- [ ] Readability: Flesch 60-70, paragraphs 2-4 sentences
- [ ] Word count: Meets or exceeds top-ranking competitor length
- [ ] Internal links: 2-5 contextual links
- [ ] External links: 2-3 authoritative sources

### Auto-Fix vs. User-Decision Required
**Auto-fix (apply immediately):**
- Character limit violations in title/meta
- Missing alt text
- Keyword density rebalancing (<0.5% or >3%)
- Heading hierarchy errors

**Require user decision:**
- H1 complete rewording
- Significant tone shifts
- Claim strength modifications (weakening hedged claims)
- Adding/removing sections

## MODULE 4: GEO CONTENT OPTIMIZER
*Source: aaron-he-zhu/seo-geo-claude-skills — geo-content-optimizer + zubair-trabzada/geo-seo-claude — geo-content*

### Trigger
User wants content to be cited by ChatGPT, Perplexity, Google AI Overviews, Claude, or Gemini.

### Core GEO Research Finding
AI models preferentially extract passages that are:
- **134-167 words** (optimal passage length)
- **Self-contained** (understandable without surrounding context)
- **Fact-rich** (3-5 specific statistics per 500 words)
- **Direct-answer format** (opens with "X is..." or "X refers to...")
- **Structurally clean** (short paragraphs, clear headings)

### 6-Technique GEO Optimization Framework

**Technique 1: Answer Block Creation (25% of citability score)**
Every major section must open with a 1-2 sentence direct answer:
```
Pattern: "[Topic] is [definition/explanation]. [Key benefit or key fact]."
Example: "IPTV (Internet Protocol Television) is a method of delivering television content over internet networks rather than traditional cable or satellite. It enables access to 5,000+ channels and VOD content from any device."
```

**Technique 2: Passage Self-Containment (25%)**
Each paragraph/block must:
- Name the subject explicitly (no "it", "they", "this" without antecedent)
- Include context so the passage makes sense in isolation
- Be 134-167 words for maximum AI extraction probability
- End with a conclusion or insight

**Technique 3: Structural Readability (20%)**
- Use question-based headings: "What Is IPTV?" not "Introduction"
- Short paragraphs: 2-4 sentences maximum
- Tables for comparisons (AI extracts tables reliably)
- Numbered lists for processes
- Bullet lists for features/benefits

**Technique 4: Statistical Density (15%)**
Every 500 words must contain 3-5 of:
- Percentages with context: "reduces setup time by 73%"
- Dollar amounts: "costs $9.99/month vs $120+/month for cable"
- Named studies: "According to Statista 2026..."
- Specific quantities: "includes 8,000+ channels across 50 countries"
- Dates/timeframes: "as of Q1 2026"

**Technique 5: Uniqueness & Original Data (10%)**
- Include at least one fact/data point NOT available elsewhere
- Conduct original comparisons, original testing, original analysis
- Cite first-party data when available
- Create proprietary frameworks or methodologies

**Technique 6: FAQ Schema + Structured Q&A**
Format every FAQ answer as a standalone GEO block:
```
Q: [Natural language question from PAA or searcher intent]
A: [25-50 word direct answer]. [Supporting detail]. [Supporting detail]. [Context or implication]. [One memorable, citable fact or statistic].
```

### GEO Content Scoring Rubric

| Category | Weight | Scoring Criteria |
|----------|--------|-----------------|
| Answer Block Quality | 30% | Direct answers in first 1-2 sentences per section |
| Passage Self-Containment | 25% | Blocks understandable without surrounding context |
| Structural Readability | 20% | Question headings, short paragraphs, tables/lists |
| Statistical Density | 15% | 3-5 specific stats per 500 words |
| Uniqueness & Original Data | 10% | First-party or exclusive information |

**Score interpretation:**
- 85-100 = Exceptional AI citation candidate
- 70-84 = Solid foundation, minor improvements needed
- 55-69 = Multiple gaps, moderate rewrite needed
- Below 55 = Significant issues, major rework required

### BEFORE/AFTER GEO Optimization Example

**BEFORE (Low GEO score):**
> "IPTV is a great way to watch TV. It offers many channels and features that people enjoy. It's much better than cable in many ways."

**AFTER (High GEO score):**
> "IPTV (Internet Protocol Television) delivers live TV, VOD, and catch-up content over broadband internet, eliminating cable infrastructure costs entirely. Unlike cable which averages $127/month in 2026, premium IPTV subscriptions cost $10-30/month while providing access to 5,000-20,000+ channels. Independent tests by StreamingComparisons.com (2026) show 99.7% uptime reliability on providers using tier-1 CDN networks."

## MODULE 5: TECHNICAL SEO AUDITOR
*Source: AgriciDaniel/claude-seo — seo-technical + zubair-trabzada/geo-seo-claude — geo-technical*

### Trigger
User wants a technical SEO audit, crawlability check, or performance analysis.

### 9-Category Technical Audit Framework (100-Point Scale)

#### Category 1: Crawlability (15 pts)
- robots.txt validity and configuration
- **AI crawler allowance** (see AI Crawler Matrix below)
- XML sitemap presence, format, and accuracy
- Crawl depth (important pages within 3 clicks of homepage)
- noindex usage (check for accidental indexation blocks)
- Crawl budget optimization for large sites

#### Category 2: Indexability (12 pts)
- Canonical tag implementation (self-referencing + cross-page)
- Duplicate content detection (parameterized URLs, www vs non-www, trailing slash)
- Pagination handling (rel=next/prev deprecated → check for canonicalization)
- Hreflang implementation for multi-language sites
- Index bloat (unnecessary pages consuming crawl budget)

#### Category 3: Security (10 pts)
- HTTPS enforcement (301 redirect from HTTP)
- Security headers: HSTS, CSP, X-Frame-Options, X-Content-Type-Options
- Mixed content detection (HTTP assets on HTTPS page)
- SSL certificate validity and expiry

#### Category 4: URL Structure (8 pts)
- Clean URLs (no session IDs, no unnecessary parameters)
- Logical hierarchy (/category/subcategory/page)
- Redirect chains (max 1 redirect hop; chains of 3+ are problematic)
- URL length (under 75 characters preferred)
- Canonical parameter handling

#### Category 5: Mobile Optimization (10 pts)
- Responsive design implementation
- Tap targets (minimum 48x48px, 8px spacing)
- Font sizes (minimum 16px body text)
- Content parity between mobile and desktop
- Mobile-first indexing compliance

#### Category 6: Core Web Vitals (15 pts)
| Metric | Good | Needs Improvement | Poor |
|--------|------|-------------------|------|
| LCP (Largest Contentful Paint) | <2.5s | 2.5-4s | >4s |
| INP (Interaction to Next Paint) | <200ms | 200-500ms | >500ms |
| CLS (Cumulative Layout Shift) | <0.1 | 0.1-0.25 | >0.25 |

**CRITICAL NOTE:** FID (First Input Delay) was fully deprecated on September 9, 2024. All references must use INP. Never use FID in audits.

#### Category 7: Server-Side Rendering (15 pts) — CRITICAL FOR GEO
**This is the most important GEO technical factor.**
AI crawlers (GPTBot, ClaudeBot, PerplexityBot) do NOT execute JavaScript.
- Detect if critical content is JavaScript-rendered vs server-rendered
- Check raw HTML source vs rendered HTML for content parity
- Identify JavaScript-rendered navigation, headings, or main content
- Verify meta tags are in raw HTML, not injected by JavaScript
- **Hard rule:** If main content is JS-rendered → AI crawlers cannot see it → GEO score severely impacted

#### Category 8: Page Speed & Server (15 pts)
- TTFB (Time to First Byte): Target <200ms
- Server response time
- Image optimization (WebP/AVIF format, proper sizing, lazy loading)
- CSS/JS minification and compression (Brotli/gzip)
- Caching headers (Cache-Control, ETag)
- CDN implementation
- Resource hints (preload, prefetch, preconnect)

#### Category 9: AI Crawler Access Matrix (Bonus — Critical for GEO)
| AI Crawler | User-Agent Token | Default Access |
|-----------|-----------------|----------------|
| OpenAI ChatGPT | GPTBot | Block/Allow |
| Anthropic Claude | ClaudeBot | Block/Allow |
| Perplexity AI | PerplexityBot | Block/Allow |
| Google Gemini | Google-Extended | ⚠️ Blocks training ONLY, NOT Search |
| Google Search | Googlebot | Allow |
| Bing / Copilot | Bingbot | Allow |
| Common Crawl | CCBot | Block/Allow |
| Apple | Applebot | Allow |

**CRITICAL DISTINCTION:** Blocking `Google-Extended` prevents Gemini training use but does NOT affect Google Search indexing or AI Overviews.

### JavaScript SEO (December 2025 Updates)
- Conflicting canonical tags between raw HTML and JS-injected versions → Google may use either
- `noindex` in raw HTML may still be honored even if JavaScript removes it
- Critical for SSR verification: Always check raw HTML source vs rendered source

## MODULE 6: SCHEMA & STRUCTURED DATA ENGINE

### Trigger

### Core Principle
"Structured data is the primary machine-readable signal that tells AI systems what an entity IS and how it connects to other entities." — geo-schema SKILL.md

### Format Priority
1. **JSON-LD** — Strongly preferred (Google-recommended, easiest to maintain)
2. **Microdata** — Legacy, migrate to JSON-LD
3. **RDFa** — Legacy, migrate to JSON-LD
**IMPORTANT:** Fetch raw HTML (not rendered) to detect JSON-LD in `<head>` blocks.

### Active Schema Types for GEO + SEO

#### Organization Schema (Highest Priority for GEO)
```json
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "[Company Name]",
  "url": "[Website URL]",
  "logo": "[Logo URL]",
  "description": "[25-50 word description optimized for AI extraction]",
  "foundingDate": "[Year]",
  "sameAs": [
    "[Wikipedia URL]",
    "[Wikidata URL]",
    "[LinkedIn URL]",
    "[YouTube URL]",
    "[Twitter/X URL]",
    "[Facebook URL]",
    "[Industry Directory URL]"
  ],
  "contactPoint": {
    "@type": "ContactPoint",
    "contactType": "customer service",
    "email": "[email]"
  }
}
```

```json
{
  "@context": "https://schema.org",
  "@type": "Article",
```

## MODULE 8: CONTENT QUALITY GATE (CORE-EEAT 80-Point Framework)
*Source: aaron-he-zhu/seo-geo-claude-skills — content-quality-auditor*

### Trigger
User asks "is this ready to publish?", wants a content quality check, or after content generation.

### The CORE-EEAT Framework

This is an 80-item evaluation system. Each item scores: **Pass (10 pts) | Partial (5 pts) | Fail (0 pts)**

There are **3 VETO GATES** — failing these blocks publication regardless of total score.

#### CORE Dimension (40 items across 4 sub-categories)

**C — Contextual Clarity (10 items)**
- C01: Intent Alignment ⚡ **VETO GATE** — Content matches dominant search intent
- C02: Direct Answer — Opening delivers clear answer within first 150 words
- C03: Scope Definition — Topic boundaries clearly defined
- C04: Audience Fit — Language and depth match target audience
- C05: Jargon Handling — Technical terms defined when introduced
- C06: Conclusion Clarity — Ending reinforces key message
- C07: Ambiguity Elimination — No vague or undefined claims
- C08: Example Usage — Abstract concepts supported with concrete examples
- C09: Counter-consideration — Acknowledges alternative views where appropriate
- C10: Actionability — Reader knows what to do with the information

**O — Organization (10 items)**
- O01: Logical Flow — Sections build on each other progressively
- O02: Heading Hierarchy — H1→H2→H3 followed without skipping
- O03: Intro-Body-Conclusion — Clear three-part structure
- O04: Paragraph Unity — Each paragraph covers one idea
- O05: Transition Quality — Smooth connections between sections
- O06: Scanability — Key points findable via headings and bullets
- O07: List Usage — Bullet/numbered lists where appropriate
- O08: Table Usage — Comparative data in table format
- O09: Visual Breaks — No walls of text (3+ paragraphs without break)
- O10: CTA Placement — Clear calls-to-action at appropriate points

**R — Referenceability (10 items)**
- R01: Source Citations — Claims backed by named, verifiable sources
- R02: Data Recency — Statistics are current (within 2 years preferred)
- R03: Authority Sources — Links to .gov, .edu, or recognized industry authorities
- R04: Quote Attribution — All quotes attributed to named individuals
- R05: Data Accuracy — Numbers and statistics are verifiable
- R06: Source Diversity — Multiple independent sources, not one provider
- R07: Internal Links — 2-5 relevant internal page links
- R08: External Links — 2-3 authoritative external sources
- R09: Link Anchor Text — Descriptive anchors (not "click here")
- R10: Content Consistency ⚡ **VETO GATE** — No contradictions within content

**E — Exclusivity (10 items)**
- E01: Original Insight — At least one point not available in competing content
- E02: Original Data — First-party research, case study, or unique analysis
- E03: Unique Angle — Different perspective from existing top-ranking content
- E04: Proprietary Framework — Own methodology or process
- E05: Expert Opinion — Quotes or insights from named experts
- E06: Case Study — Real example with specific outcomes
- E07: Original Imagery — Custom images/diagrams vs. stock
- E08: Depth Advantage — More comprehensive than any single competitor
- E09: Recency Advantage — More current than top competitors
- E10: Format Differentiation — Unique presentation format

#### EEAT Dimension (40 items across 4 sub-categories)

**Experience (10 items)**
- Exp01: First-Hand Knowledge — Direct involvement evidence
- Exp02: Specific Outcomes — Results with actual metrics
- Exp03: Process Documentation — How the author did something
- Exp04: Personal Perspective — Unique viewpoint from experience
- Exp05: Before/After Context — Transformation evidence
- Exp06: Failure Disclosure — Honest acknowledgment of what didn't work
- Exp07: Time Investment — Duration of experience demonstrated
- Exp08: Iteration Evidence — Improvement through repeated attempts
- Exp09: Tool/Resource Usage — Specific tools or resources used
- Exp10: Sensory Details — Descriptive, lived-experience language

**Expertise (10 items)**
- Exp11: Technical Depth — Domain-specific precision
- Exp12: Terminology Mastery — Correct use of specialized vocabulary
- Exp13: Methodology Clarity — Clear explanation of approach
- Exp14: Nuance Recognition — Acknowledges complexity and edge cases
- Exp15: Contrarian Awareness — Knows and addresses opposing views
- Exp16: Trend Awareness — References current developments
- Exp17: Limitation Acknowledgment — Honest about what is not known
- Exp18: Credential Display — Qualifications, certifications, or experience shown
- Exp19: Cross-Domain Connection — Relates topic to adjacent expertise
- Exp20: Predictive Reasoning — Evidence-based projections

**Authority (10 items)**
- Auth01: External Recognition — Media mentions, awards, citations
- Auth02: Inbound Links — Evidence of being linked to by others
- Auth03: Community Presence — Active in relevant communities
- Auth04: Peer Validation — Reviewed or endorsed by peers
- Auth05: Publication History — Track record of authoritative content
- Auth06: Topical Breadth — Coverage of adjacent topics in domain
- Auth07: Industry Participation — Conference speaking, contributions
- Auth08: Collaborative Work — Co-authored with other authorities
- Auth09: Institutional Affiliation — Association with recognized organizations
- Auth10: Wikipedia Presence — Entity or author referenced on Wikipedia

**Trustworthiness (10 items)**
- Trust01: Contact Information — Readily accessible
- Trust02: Privacy Policy — Up-to-date and accessible
- Trust03: About Page — Clear about who is behind the content
- Trust04: Disclosure Statements ⚡ **VETO GATE** — Affiliations/sponsorships clearly labeled
- Trust05: HTTPS Security — Site uses HTTPS
- Trust06: Editorial Standards — Process for accuracy described
- Trust07: Business Model Disclosure — How the site makes money is clear
- Trust08: Review Signals — Third-party reviews visible
- Trust09: Update Transparency — Content update dates shown
- Trust10: Error Correction Policy — Mistakes acknowledged and corrected

## MODULE 11: llms.txt GENERATOR & ANALYZER
*Source: zubair-trabzada/geo-seo-claude — geo-llmstxt*

### What Is llms.txt?
Proposed by Jeremy Howard (2024), llms.txt functions like robots.txt for AI systems — a machine-readable file helping AI understand site structure, purpose, and key pages.

**Key Benefits:**
- Faster AI comprehension of site purpose
- Controlled narrative about your brand (reduces AI hallucination)
- Higher citation accuracy in AI responses
- Early-adopter advantage: fewer than 5% of sites have llms.txt

### llms.txt Format Requirements
Location: `https://yourdomain.com/llms.txt`

```markdown
# [Site/Business Name]

> [One-sentence description, under 200 characters, optimized for AI extraction]

## Docs
- [Page Title](https://example.com/page): [10-30 word description, specific and factual]
- [Page Title](https://example.com/page): [Description]

## Blog
- [Post Title](https://example.com/blog/post): [Description]

## Key Facts
- Founded: [Year]
- Services: [Specific services]
- Pricing: [Price range or model]
- Coverage: [Geographic or topic coverage]
- Contact: [Email or contact method]

## Optional
- [Supporting page title](URL): [Description]
```

### Quality Standards for Descriptions
**BAD:** "React guide." — Too vague, provides no AI value.
**GOOD:** "Comprehensive 3,000-word guide to React Server Components with working code examples for Next.js 15." — Specific, factual, useful for AI comprehension.

*Source: aaron-he-zhu/seo-geo-claude-skills — backlink-analyzer + domain-authority-auditor (CITE framework)*

### Trigger
User wants to analyze backlink profile, check domain authority, or build link building strategy.

### CITE Domain Rating Framework (40-item Assessment)

**C — Citation Potential (10 items)**
- C01: Content linkability (is there inherently linkable content: studies, tools, guides?)
- C02: Original data presence (proprietary data others want to cite)
- C03: Visual asset library (infographics, charts others can embed)
- C04: Expert contributions (named experts others reference)
- C05: Tool/resource availability (free tools others link to)
- C06: Media outreach potential (newsworthy content)
- C07: Academic citability (scholarly or research-grade content)
- C08: Industry report quality (annual reports or trend analyses)
- C09: Case study richness (detailed outcomes worth referencing)
- C10: Quote generation (memorable, shareable statements)

**I — Infrastructure (10 items)**
- I01: Domain age and history
- I02: Technical health score (from Technical Audit)
- I03: Page speed performance
- I04: Mobile optimization
- I05: SSL/security implementation
- I06: Site architecture depth
- I07: Internal link structure quality
- I08: Schema markup completeness
- I09: Content freshness/update frequency
- I10: Content volume and depth

**T — Trust Signals (10 items)**
- T01: About page completeness
- T02: Contact information accessibility
- T03: Privacy policy and terms
- T04: Named authorship throughout site
- T05: Physical address presence
- T06: Phone number availability
- T07: Business registration/verification signals
- T08: Press mentions and media coverage
- T09: Awards or recognitions
- T10: Professional certifications displayed

**E — Entity Recognition (10 items)**
- E01: Wikipedia entity page existence
- E02: Wikidata Q-number assigned
- E03: Google Knowledge Panel present
- E04: Social media profile completeness and consistency
- E05: NAP consistency across web
- E06: sameAs links in Organization schema
- E07: Brand name search volume (branded searches)
- E08: Industry directory listings
- E09: Chamber of commerce or trade association membership
- E10: YouTube channel with consistent branding

### Backlink Analysis Checklist
- Total referring domains (quality over quantity)
- Domain authority distribution (% of links from DA>40 sites)
- Anchor text distribution (branded vs. exact match vs. generic)
- Link velocity (rate of new links acquired)
- Toxic link identification (spammy, irrelevant, or link-farm links)
- Competitor backlink comparison
- Link gap (links competitors have that you don't)

## MODULE 19: AEO / FEATURED SNIPPET ENGINE
*Sources: All three repositories — cross-cutting AEO techniques*

### Trigger
User wants to win featured snippets, appear in People Also Ask, or capture AI Overview citations.

### Featured Snippet Types and Formats

**Type 1: Paragraph Snippet**
Target: Definition queries, "what is", "who is", "why is"
Format: 40-60 word direct answer paragraph
```
Template:
[Term] is [definition]. [Key characteristic]. [Why it matters].
Example: "IPTV is a television delivery method using internet protocols instead of traditional cable or satellite infrastructure. It enables on-demand access to thousands of channels through any internet-connected device, typically at 70-80% lower cost than cable TV subscriptions."
```

**Type 2: List Snippet (Numbered)**
Target: "How to", step-by-step processes, ranked lists
Format: 5-8 numbered items, each 5-12 words
```
Template:
H2: How to [Do X]
1. [Action verb] + [specific detail]
2. [Action verb] + [specific detail]
...
```

**Type 3: List Snippet (Bulleted)**
Target: "Types of", "examples of", "benefits of"
Format: 6-10 bullet points
```
Template:
H2: [Number] Types/Benefits/Examples of [Topic]
• [Item]: [Brief description]
• [Item]: [Brief description]
...
```

**Type 4: Table Snippet**
Target: Comparison queries, "X vs Y", "pricing", specifications
Format: 2-4 column table, 4-8 rows
```
| [Category] | [Option A] | [Option B] |
|------------|-----------|-----------|
| [Factor] | [Value] | [Value] |
```

### People Also Ask (PAA) Targeting Strategy
1. Search target keyword and record all PAA questions
2. Group PAA questions by subtopic
3. Create H2 or H3 headings that exactly match the question wording
4. Answer immediately after the heading (within 50 words)
5. Expand with supporting detail (to capture the "expand" click)
6. Structure answer as: [Direct answer] + [Supporting fact] + [Example or implication]

### AI Overview Optimization Rules
1. **Definitions** — Every major term needs a clear, standalone definition
2. **Statistics** — Include current, specific statistics with named sources
3. **Direct answers** — Open every section with the conclusion, not the build-up
4. **Structured format** — Tables and lists extracted 60% more than prose
5. **Source authority** — Cite recognizable sources (Statista, Gartner, official bodies)
6. **Freshness** — AI Overviews strongly prefer pages with recent update dates

## MODULE 20: CONVERSION-FOCUSED CONTENT ENGINE
*Bonus module — for IPTV, digital services, and subscription business optimization*

### Trigger
User wants content that not only ranks but also converts visitors to customers.

### Conversion Psychology for Digital Services (IPTV/Streaming)

**The 5-Stage Buyer Journey:**
1. **Awareness** — "what is IPTV", "how does streaming work"
2. **Consideration** — "best IPTV services", "IPTV vs cable", "IPTV reviews"
3. **Decision** — "cheap IPTV subscription", "buy IPTV plan", "IPTV trial"
4. **Retention** — "how to set up IPTV", "IPTV troubleshooting"
5. **Advocacy** — "IPTV referral", "IPTV affiliate program"

### High-Converting Content Structures

**Landing Page Formula (for IPTV/Digital Services):**
```
[HEADLINE]: [Primary Benefit] + [Specific Number] + [Risk Reduction]
Example: "Watch 12,000+ Channels for $12/Month — 24-Hour Free Trial, No Credit Card"

[HERO SECTION]:
- Primary benefit statement (10 words max)
- 3 key differentiators (bullet points)
- CTA button: "[Action] [Benefit]" (e.g., "Start Free Trial Now")
- Social proof snippet: "Join 50,000+ subscribers"

[TRUST SECTION]:
- Number of channels/content
- Compatibility (devices/platforms)
- Uptime percentage
- Customer rating

[FEATURE BREAKDOWN]:
Per feature: [What it is] + [Specific benefit] + [Who it's for]

[PRICING SECTION]:
- Plans side-by-side (monthly / 3-month / 6-month / annual)
- Best value badge on middle tier
- Money-back guarantee prominently displayed
- FAQ objection handling (5 most common objections)

[SOCIAL PROOF]:
- Reviews with specifics ("I switched from cable, saving $90/month")
- Star rating + review count
- Logos of supported devices/platforms

[FINAL CTA]:
- Repeat primary CTA
- Urgency element if applicable
- Risk reversal (money-back guarantee, free trial)
```

### Conversion-Boosting Copy Patterns

**Pattern 1: Specificity Over Vagueness**
- BAD: "Many channels available"
- GOOD: "12,000+ live channels including 200+ sports networks"

**Pattern 2: Value Comparison**
- BAD: "Affordable IPTV service"
- GOOD: "From $9.99/month — that's $8/month less than Netflix alone"

**Pattern 3: Risk Reduction**
- BAD: "Try our service"
- GOOD: "Start your 24-hour free trial — no credit card required"

**Pattern 4: Social Proof + Specificity**
- BAD: "Customers love us"
- GOOD: "4.8/5 from 2,400+ verified reviews — 97% renewal rate"

**Pattern 5: Urgency + Scarcity (when applicable)**
- BAD: "Limited time offer"
- GOOD: "Black Friday pricing ends in 47 hours — save 60% on 12-month plan"

### SEO + Conversion Keyword Mapping
| Intent Stage | Keyword Pattern | Content Type | Primary CTA |
|-------------|----------------|--------------|-------------|
| Awareness | "what is IPTV" | Informational blog | "Learn more" → email capture |
| Consideration | "best IPTV service 2026" | Comparison article | "See plans" → pricing page |
| Decision | "buy IPTV subscription" | Landing page | "Start trial" / "Buy now" |
| Local | "IPTV service [country]" | Local landing page | "Get started" |
| Objection | "is IPTV legal" | FAQ/blog | Trust signals → CTA |
